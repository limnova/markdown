<threat_model>
## Trust Boundaries

- `design contract -> CSS/classes`: the UI must communicate blocked and safe states accurately so users do not mistake external links for workspace content.
- `workspace payload -> rendered rows`: entry kinds returned by the backend cross into presentational components and must remain non-destructive.

## STRIDE Register

| threat_id | category | component | disposition | mitigation_plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | external-link visual treatment | mitigate | Use explicit labels, muted styling, and disabled interaction affordances so out-of-root rows cannot be mistaken for normal workspace items. |
| T-02-02 | Information Disclosure | broken-path display | mitigate | Show only the remembered path and recovery CTA; do not auto-browse or expand invalid paths. |
| T-02-03 | Tampering | boundary banner copy | mitigate | Use the exact rejection message in a dedicated inline banner component so later integrations do not improvise weaker wording. |
</threat_model>
