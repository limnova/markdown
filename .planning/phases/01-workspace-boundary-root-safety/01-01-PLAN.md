<threat_model>
## Trust Boundaries

- `webview -> Rust workspace command`: selected or restored path strings cross from the UI into the native layer.
- `workspace root -> filesystem children`: entries inside the chosen folder may resolve outside the workspace through symlinks, junctions, or shortcuts.
- `capability file -> runtime permissions`: plugin and command exposure in `default.json` defines what the webview can reach.

## STRIDE Register

| threat_id | category | component | disposition | mitigation_plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `load_workspace` path validation | mitigate | Canonicalize the candidate root in Rust and never rely on raw string prefix checks. |
| T-01-02 | Elevation of Privilege | main-window capability surface | mitigate | Add only `dialog:default` and `store:default`; do not grant wide filesystem plugin scopes in Phase 1. |
| T-01-03 | Information Disclosure | root child classification | mitigate | Resolve children against the canonical root and mark out-of-root targets as `external-link` with `interactive: false`. |
</threat_model>
